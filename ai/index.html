AI Chat Widget

An embeddable AI receptionist for any existing website. We install it, tune it, and run it as a managed SaaS. Sells on its own or as the entry tier of the founding plan. Brainstorming doc — not yet customer-facing.

Drafted: May 2026 Status: Internal brainstorm Owners: Builder & Sales partner

The Product

An AI chat widget that drops into any existing website (one line of script tag). Backed by a RAG knowledge base the customer controls — they upload their docs, FAQs, product info, or we crawl their existing site. The agent's personality and purpose are configurable per customer: pure customer-support Q&A, lead capture (email/phone/SMS), booking concierge, or hybrid.

The infrastructure is already built. What we sell is:

  1. Install & tune. One-hour professional setup on the customer's existing site. We ingest their knowledge sources, write the system prompt, calibrate tone, set lead-routing rules.
  2. Hosted SaaS. Customer pays monthly. We host the agent backend, manage tokens, handle updates, monitor performance.
  3. Managed knowledge base. Monthly refresh of the customer's content; new docs go in, stale ones come out.

Compared to Intercom Fin, Tidio, Chatbase, Crisp, or rolling-your-own ChatGPT plugin: we differentiate on the install + tune + run service layer. The widget itself is commoditized; the white-glove integration isn't.

Pricing Rationale

Token economics

At realistic usage per tier, using Claude Haiku or GPT-4o-mini for retrieval-augmented responses:

  • Hook (100 convos/mo): ~$2/mo in API costs. Gross margin ~96%.
  • Standard (300 convos/mo): ~$6/mo. Gross margin ~94%.
  • Pro (1,500 convos/mo soft cap): ~$25/mo. Gross margin ~87%.

Even with worst-case heavy usage hitting Pro's overage tier, the $0.05/conversation overage rate covers our cost (~$0.015–$0.02 per conversation) with margin to spare.

Comparison to commoditized alternatives

  • Tidio / Crisp / Chatra: $19–$49/mo for a chat widget with limited AI. The customer self-installs, self-tunes. We're priced similarly but include the install + tune service.
  • Intercom Fin: $0.99 per resolution, sits on top of Intercom ($75+/mo) — fast adds up. We're flat-fee, predictable.
  • Chatbase / Voiceflow / build-your-own: $29–$399/mo for self-serve RAG agents. Customer does all the work themselves. We do it for them.

The differentiation, in one sentence

We are the only one in this band that installs and tunes the widget on the customer's existing site as part of the price. That hour of work is the entire reason we can charge $49/mo and not be priced against $19/mo self-serve alternatives.

Billing & Commitment

  • Setup fee charged on signup (one-time, non-refundable): $199 / $299 / $499 by tier
  • Monthly recurring billed via Stripe on the signup anniversary
  • Annual prepay option saves ~17%: $489 / $989 / $1,989 per year
  • Customers can upgrade or downgrade at any monthly renewal
  • Pro overage billed monthly at $0.05/conversation above 1,500
  • 30-day minimum on monthly subscriptions; cancel any time after

How This Fits with the Founding Plan

Two ways to position this product:

Option Mechanic Tradeoff
A — Replace Starter tier The founding plan's $50/mo Starter tier (basic landing page) becomes the AI Chat Widget Hook at $49/mo. Starter today competes with Squarespace at half the price; the AI widget gives that price point a real reason to exist. Single sales pitch. Strengthens a weak tier. Customers upgrade to Standard ($100/mo) which bundles widget + website. Recommended.
B — Standalone product line Sell as a separate product, possibly on a sub-brand. Doesn't compete with the website SaaS. Cross-sell to widget customers when they later want a website rebuild. Two product lines, two pitches, two pricing pages to maintain. More cognitive load on a 2-person team and on prospects.
Recommendation: Test as a standalone product in the validation phase (next 30 days). If it converts at acceptable rates, fold into the founding plan as the new Starter tier in DRAFT v2.0 — that's the cleanest outcome. If it doesn't convert, kill it cheaply before integrating.

Pilot Path

  1. Stand up a landing page at a sub-route (e.g., chat.maks.co or maks.co/widget) with the three tiers and a Stripe Checkout signup flow. Half-day of work using existing infrastructure.
  2. Cold outreach to 10 SMBs with existing websites in your network and the cousin's. Pitch: "We install an AI receptionist on your site in an hour. $49/mo, 100 conversations included, money-back if it doesn't convert leads in 30 days."
  3. Track conversion + token usage. If 2 of 10 sign up, the model is real. If 0 sign up, the price or pitch is wrong. If >5 sign up, raise the price next round.
  4. Decide Option A vs. B based on the data. If the widget pulls leads who also want websites, A. If it pulls a separate customer base entirely, B.
  5. Codify the install playbook. Document the 1-hour setup as a reproducible checklist so it stays at 1 hour as volume grows. Without this, install time inflates and margins collapse.

Things to Pressure-Test

  • Token-cost variance under abuse — a single customer with a public widget could absorb thousands of conversations from spammers or curious developers. Hard cap with soft warnings is critical. Cloudflare Turnstile or rate-limiting per visitor IP needs to be in place from day one.
  • Differentiation against free / freemium alternatives — "we install it for you" is real but only persuasive to buyers who don't want to DIY. Some prospects will hear $49/mo and ask why they shouldn't just use Tidio's free tier. The answer is the install service + tuning + ongoing knowledge management — make sure that's front-and-center in the pitch, not buried.
  • B2B SMB sales economics at $49/mo — payback math says you need ~6 months to recoup CAC. At $49 that's $300 in lifetime value before contribution, against probably $100–$300 in our acquisition time per customer. Cold outreach has to be cheap (network, referrals, content) or we push price up to $99 to accelerate payback.
  • 1-hour install at scale — fine for the first 10 customers, brittle at 50. The codified playbook in the pilot path is the answer; without it, install drift kills the unit economics.
  • Token-model stability — if Claude or OpenAI raise prices or deprecate the cheap models, gross margin compresses. We should be model-agnostic in the backend so we can swap providers as economics shift.
  • Brand confusion vs. founding plan — if this lives at pricing.maks.co/ai while the main plan is at pricing.maks.co, customers may not know which to pick. Either keep them visually distinct sub-brands or fold this into the main plan quickly.
